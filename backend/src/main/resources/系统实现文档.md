# 校园论坛智能审核系统实现文档

## 1. 系统架构

### 1.1 整体架构

```
用户提交内容 → SpringBoot后端 → Python Flask AI服务 → 审核决策
                    ↓                    ↓
              本地敏感词兜底        DFA敏感词过滤
                                   LSTM情感分析
                                   综合决策引擎
```

### 1.2 核心模块

- **SpringBoot后端**：处理用户请求，调用AI服务进行内容审核
- **Python AI服务**：提供敏感词检测和情感分析功能
- **审核决策引擎**：根据敏感词检测和情感分析结果做出审核决策
- **人工复核系统**：处理需要人工审核的内容

## 2. 功能实现

### 2.1 内容审核流程

1. **内容提交**：用户提交帖子或评论
2. **纯文本提取**：从富文本内容中提取纯文本
3. **本地敏感词检查**：作为兜底措施
4. **AI服务审核**：调用Python AI服务进行深度审核
5. **审核决策**：
   - `ALLOW` → 直接发布
   - `BLOCK` → 拒绝发布
   - `MANUAL_REVIEW` → 标记待审核
6. **结果反馈**：向用户返回审核结果

### 2.2 敏感词过滤

- 使用DFA（确定有限自动机）算法实现高效敏感词检测
- 支持多类别词库：政治、色情、暴恐、违法等
- 支持词库热更新

### 2.3 情感分析

- 使用LSTM（长短期记忆网络）模型进行情感分析
- 支持三分类：正向、负向、中性
- 模型训练基于多个情感分析数据集

### 2.4 人工复核

- 待审核内容列表管理
- 批量审核操作
- 审核历史记录

## 3. 技术实现

### 3.1 SpringBoot后端

#### 核心文件
- `TopicServiceImpl.java`：实现帖子和评论的审核逻辑
- `AiServiceUtils.java`：封装AI服务调用
- `ModerationAdminController.java`：提供人工审核接口

#### 关键方法
- `createTopic()`：创建帖子并进行审核
- `createComment()`：创建评论并进行审核
- `listPendingReviewTopics()`：获取待审核帖子
- `listPendingReviewComments()`：获取待审核评论
- `approveTopic()`：批准帖子
- `rejectTopic()`：拒绝帖子
- `approveComment()`：批准评论
- `rejectComment()`：拒绝评论

### 3.2 Python AI服务

#### 核心文件
- `app.py`：Flask RESTful API服务
- `moderation.py`：审核引擎
- `sensitive_words.py`：敏感词过滤器
- `inference.py`：情感分析模型推理

#### 关键接口
- `POST /moderate`：内容审核
- `POST /moderate/batch`：批量审核
- `GET/POST /moderation/config`：审核策略配置
- `POST /sensitive_words/check`：敏感词检测
- `GET /sensitive_words/stats`：词库统计

## 4. 审核策略

### 4.1 敏感词策略
- **严重类别**（暴恐/违法/反动）→ 直接拦截
- **风险分数 ≥ 60** → 直接拦截
- **风险分数 40-59** → 人工审核

### 4.2 情感分析策略
- **负向概率 ≥ 85%** → 直接拦截
- **负向概率 70-84%** → 人工审核

## 5. 部署指南

# 启动服务
cd src

## 6. 接口说明

### 6.1 AI服务接口

#### 内容审核
- **URL**: `/moderate`
- **方法**: POST
- **参数**:
  - `text`: 待审核文本
  - `content_type`: 内容类型 (topic/comment/username/signature)
- **返回**:
  - `success`: 操作是否成功
  - `data`: 审核结果
    - `action`: 审核决策 (ALLOW/BLOCK/MANUAL_REVIEW)
    - `reason`: 审核原因
    - `sentiment_score`: 情感分析分数
    - `sensitive_words`: 检测到的敏感词

### 6.2 管理接口

#### 待审核帖子
- **URL**: `/api/admin/moderation/pending-topics`
- **方法**: GET
- **参数**:
  - `page`: 页码
  - `size`: 每页数量
- **返回**: 待审核帖子列表

#### 批准帖子
- **URL**: `/api/admin/moderation/approve-topic`
- **方法**: GET
- **参数**:
  - `tid`: 帖子ID
- **返回**: 操作结果

- **智能分析**：LSTM模型进行情感分析

## 10. 总结

校园论坛智能审核系统通过结合敏感词过滤和情感分析技术，实现了对用户内容的自动审核，有效识别和拦截不良内容。系统采用双层防控机制，既保证了审核的准确性，又提供了人工复核的渠道，确保内容审核的公正性和灵活性。

系统的实现不仅提高了论坛内容的质量，也减轻了人工审核的负担，为校园论坛的健康发展提供了有力的技术支持。
