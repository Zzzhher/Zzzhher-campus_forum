# 校园论坛内容审核策略文档

## 一、概述

本文档定义了校园论坛系统的内容审核策略，实现"AI初筛 + 人工复核"的双层防控体系。

## 二、审核架构

```
用户提交内容
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│                    第一层：敏感词过滤 (DFA算法)               │
│  - 严重敏感词 → 直接拦截 (BLOCK)                            │
│  - 一般敏感词 → 标记风险分数                                 │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│                    第二层：AI情感分析 (LSTM)                 │
│  - 三分类：正向 / 负向 / 中性                               │
│  - 输出置信度和概率分布                                     │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│                    第三层：综合决策引擎                      │
│  ├─ ALLOW (通过)        → 直接发布                          │
│  ├─ BLOCK (拦截)        → 拒绝发布 + 记录日志               │
│  └─ MANUAL_REVIEW (人工审核) → 标记待审核 + 进入人工队列     │
└─────────────────────────────────────────────────────────────┘
```

## 三、敏感词策略

### 3.1 词库分类

| 类别 | 文件                          | 严重程度 | 处理方式 |
| ---- | ----------------------------- | -------- | -------- |
| 暴恐 | 暴恐词库.txt                  | 严重(3)  | 直接拦截 |
| 违法 | 涉枪涉爆.txt, 非法网址.txt    | 严重(3)  | 直接拦截 |
| 反动 | 反动词库.txt                  | 严重(3)  | 直接拦截 |
| 色情 | 色情词库.txt, 色情类型.txt    | 一般(2)  | 风险评分 |
| 政治 | 政治类型.txt, 新思想启蒙.txt  | 一般(2)  | 风险评分 |
| 广告 | 广告类型.txt                  | 一般(2)  | 风险评分 |
| 贪腐 | 贪腐词库.txt                  | 一般(2)  | 风险评分 |
| 疫情 | COVID-19词库.txt              | 一般(2)  | 风险评分 |
| 民生 | 民生词库.txt                  | 一般(2)  | 风险评分 |
| 其他 | 其他词库.txt, 补充词库.txt 等 | 一般(2)  | 风险评分 |

### 3.2 风险分数计算

```
风险分数 = Σ(命中词严重程度 × 20)
上限: 100分
```

### 3.3 敏感词处理阈值

| 风险分数 | 处理方式                 |
| -------- | ------------------------ |
| ≥ 60分   | 直接拦截 (BLOCK)         |
| 40-59分  | 人工审核 (MANUAL_REVIEW) |
| < 40分   | 进入情感分析阶段         |

## 四、情感分析策略

### 4.1 模型输出

- **三分类**: 正向 / 负向 / 中性
- **置信度**: 0-1之间的概率值
- **概率分布**: 三个类别的概率分布

### 4.2 情感分析阈值

| 场景     | 负向概率 | 处理方式 |
| -------- | -------- | -------- |
| 普通内容 | ≥ 85%    | 直接拦截 |
| 普通内容 | 70%-84%  | 人工审核 |
| 普通内容 | < 70%    | 通过     |

## 五、内容类型差异

| 内容类型      | 审核严格度 | 说明              |
| ------------- | ---------- | ----------------- |
| 帖子(TOPIC)   | 标准       | 标题+内容综合审核 |
| 评论(COMMENT) | 标准       | 评论内容审核      |
| 用户名        | 严格       | 禁止任何敏感词    |
| 个性签名      | 标准       | 常规审核          |

## 六、审核决策矩阵

| 敏感词风险 | 情感分析   | 处理方式      |
| ---------- | ---------- | ------------- |
| 严重违规   | 任意       | BLOCK         |
| ≥60分      | 任意       | BLOCK         |
| 40-59分    | 任意       | MANUAL_REVIEW |
| <40分      | 负向≥85%   | BLOCK         |
| <40分      | 负向70-84% | MANUAL_REVIEW |
| <40分      | 负向<70%   | ALLOW         |
| 无         | 负向≥85%   | BLOCK         |
| 无         | 负向70-84% | MANUAL_REVIEW |
| 无         | 其他       | ALLOW         |

## 七、人工复核流程

### 7.1 待审核队列

- 状态标记: `status = 1` (待审核)
- 对用户展示: "内容正在审核中"
- 审核人员可在后台查看待审核列表

### 7.2 审核操作

| 操作 | 效果     | 状态变更            |
| ---- | -------- | ------------------- |
| 批准 | 内容发布 | status = 0          |
| 拒绝 | 内容删除 | status = 2 (已拒绝) |

### 7.3 批量操作

支持批量批准/拒绝，提高审核效率。

## 八、审核日志

### 8.1 日志内容

```json
{
  "action": "allow/block/manual",
  "content_type": "topic/comment",
  "text": "内容摘要(前100字)",
  "sentiment": {
    "label": "正向/负向/中性",
    "confidence": 0.95,
    "probabilities": {
      "正向": 0.05,
      "负向": 0.95,
      "中性": 0.0
    }
  },
  "sensitive_words": {
    "matched": true,
    "words": ["敏感词1", "敏感词2"],
    "categories": ["色情", "广告"],
    "risk_score": 45.0,
    "is_severe": false
  },
  "reason": "审核原因说明",
  "timestamp": "2024-01-01T12:00:00"
}
```

### 8.2 日志存储

- 所有审核操作记录到数据库
- 支持按时间、用户、内容类型筛选
- 支持导出审核报表

## 九、策略调整接口

### 9.1 可配置参数

```python
{
  "negative_threshold": 0.85,        # 负向情感人工审核阈值
  "negative_block_threshold": 0.98,  # 负向情感拦截阈值
  "sensitive_risk_threshold": 40,    # 敏感词风险人工审核阈值
  "sensitive_block_threshold": 60    # 敏感词拦截阈值
}
```

### 9.2 热更新

支持不重启服务的情况下更新审核策略。

## 十、性能指标

| 指标       | 目标值     |
| ---------- | ---------- |
| 敏感词检测 | < 10ms/条  |
| 情感分析   | < 100ms/条 |
| 总审核时间 | < 200ms/条 |
| 准确率     | > 90%      |
| 误杀率     | < 5%       |

## 十一、后续优化方向

1. **模型优化**: 持续训练LSTM模型，提高情感分析准确率
2. **词库维护**: 定期更新敏感词库，添加新出现的敏感词
3. **策略调优**: 根据审核日志分析，调整阈值参数
4. **用户画像**: 建立用户信用体系，可信用户降低审核强度
5. **主动学习**: 将人工审核结果反馈到模型训练中
